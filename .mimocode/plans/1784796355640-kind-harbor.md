# Mobile Performance Plan

## Problem
Portfolio is laggy on mobile. Multiple GPU-heavy effects running simultaneously: Two Three.js canvases with MeshTransmissionMaterial, Matter.js physics with grab interaction, logo carousel re-rendering every 100ms, 5-layer backdrop-filter blur, video backgrounds, canvas animation loops, and SVG filter chains.

## User Decisions
- **Three.js glass**: KEEP on mobile (optimize instead of remove)
- **Matter.js physics**: KEEP on mobile but disable grab/mouse interaction
- **InteractiveGrid canvas**: DISABLE on mobile

---

## Changes

### 1. Optimize Three.js Glass on Mobile

**`components/FluidGlass.tsx`**
- Reduce DPR from `[1, 1.5]` to `[1, 1]` on mobile (`dpr={[1, 1]}`)
- Add `frameloop="demand"` to mobile Canvas â€” only re-render when something changes (the cube is static)
- Set `powerPreference: "default"` on mobile instead of `"high-performance"`
- In `EmbedCube` useFrame: skip FBO render + scene render when not visible (use IntersectionObserver or simple visibility check)

**`components/FluidGlassCard.tsx`**
- Pass a `mobile` prop to `FluidGlass` to control DPR and frameloop

### 2. Optimize Matter.js on Mobile (Disable Grab)

**`sections/tools.tsx`**
- Pass `grabCursor={false}` to `<Gravity>` on mobile
- Remove `onMouseMove` / `onMouseLeave` handlers on mobile (no mouse on touch devices)
- The `handleMouseMove` function does bucket detection + label animation â€” skip entirely on mobile

**`components/fancy/physics/gravity.tsx`**
- When `grabCursor={false}`, skip `MouseConstraint` creation and all mouse event listeners
- Skip `Events.on(engine, "beforeUpdate")` cursor logic when grab is disabled

### 3. Fix LogoCarousel 100ms Re-renders

**`components/ui/logo-carousel.tsx`**
- Replace `setInterval(updateTime, 100)` with `setInterval(updateTime, 1000)`
- Or better: track time with `useRef` + `requestAnimationFrame`, only call `setCurrentTime` when `Math.floor(time / cycleInterval)` changes
- This reduces React re-renders from 10/sec to ~0.25/sec per column

### 4. Disable InteractiveGrid on Mobile

**`components/InteractiveGrid.tsx`**
- Add mobile detection (`window.innerWidth < 1024`)
- On mobile: skip canvas creation, skip `mousemove` listener, skip `requestAnimationFrame` loop
- Keep the CSS `backgroundImage` grid pattern (pure CSS, no canvas needed)
- Keep `children` rendering

### 5. Optimize GradualBlur on Mobile

**`app/layout.tsx`**
- On mobile: reduce `divCount` from 5 to 2, reduce `strength` from 2 to 1
- This cuts backdrop-filter operations from 5 to 2

### 6. Optimize Video Backgrounds

**`app/layout.tsx`**
- On mobile: set `preload="none"` on both videos, or use a static poster image
- Or: only render one video on mobile (currently both are in DOM, one hidden via CSS)
- Add `playsInline` (already there) + consider `disco

### 7. Simplify SVG Filters on Mobile

**`components/GlassSurface.tsx`** (used by BottomNav)
- On mobile: skip SVG filter pipeline, use simple `backdrop-filter: blur(12px)` only
- The BottomNav is small â€” the complex feDisplacementMap chain is overkill

**`components/GlassCube.tsx`**
- On mobile: same â€” simple backdrop-filter fallback
- Note: GlassCube is in the archive (re-exported from LiquidGlassCard), not actively used in the main layout

### 8. Font Loading Optimization

**`app/fonts.ts`**
- Currently loads 9 font files eagerly. No code change needed â€” Next.js `localFont` with `display: 'swap'` is already optimal
- The fonts are local (not network requests), so this is low priority
