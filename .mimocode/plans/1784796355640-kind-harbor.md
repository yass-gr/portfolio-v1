# Mobile Performance Plan

## Problem
Portfolio app is laggy on mobile due to multiple concurrent GPU-heavy effects: Two Three.js canvases with MeshTransmissionMaterial, Matter.js physics simulation, 5-layer backdrop-filter blur, 100ms interval re-renders, canvas animation loops, and complex SVG filter chains — all running simultaneously.

## Root Causes (sorted by impact)

### 1. Two Three.js Canvases (`FluidGlass` in `FluidGlassCard`)
- **Hero** (`sections/Hero.tsx:313`) — renders `LiquidGlassCard` which embeds a `FluidGlass` cube with `MeshTransmissionMaterial` + FBO + GLTF
- **Projects** (`sections/Projects.tsx:145`) — renders `FluidGlassCard` with same setup
- `MeshTransmissionMaterial` is the most expensive Three.js material — renders scene to FBO texture every frame for refraction
- Each runs its own `requestAnimationFrame` render loop

### 2. Matter.js Physics Engine (`components/fancy/physics/gravity.tsx`)
- Full 2D physics simulation with 23+ dynamic bodies (tool pills) + 15+ static walls
- Runs at 60fps via `Runner` + `Render` + custom `requestAnimationFrame` loop
- Creates a hidden canvas, initializes `Engine`, `MouseConstraint`, multiple walls
- `gravity.tsx:363-366` — `Render.run()` + `Runner.run()` + `updateElements()` all active

### 3. LogoCarousel 100ms Re-render (`components/ui/logo-carousel.tsx:126-129`)
- `setInterval(updateTime, 100)` — updates `currentTime` state every 100ms
- Each tick triggers React re-render of all `LogoColumn` components with `AnimatePresence`

### 4. GradualBlur 5-Layer Stack (`app/layout.tsx:74-83`)
- 5 divs each with `backdropFilter: blur(Nrem)` applied to entire page bottom
- Each layer is a GPU compositing operation — 5 stacked = very expensive on mobile

### 5. Video Backgrounds (`app/layout.tsx:46-63`)
- Two `.webm` files (3.1MB + 2.7MB) both `<video loop autoPlay>` in DOM
- Even though one is hidden via CSS, both decode on mobile
- Mobile GPU has to decode video + composite with all other effects

### 6. InteractiveGrid Canvas (`components/InteractiveGrid.tsx`)
- Continuous `requestAnimationFrame` loop with `shadowBlur` (expensive)
- Global `mousemove` listener
- Idle animation with random dot movement

### 7. SVG Filter Chains (GlassSurface + GlassCube)
- Multiple `feDisplacementMap` + `feColorMatrix` + `feBlend` + `feGaussianBlur` operations
- Each applied as `backdropFilter` — extremely GPU-intensive

### 8. 9 Custom Font Files (`app/fonts.ts`)
- All loaded eagerly at layout level, no font-display optimization beyond `swap`

## Implementation Plan

### Phase 1: Disable Three.js on Mobile (Biggest Impact)

**File: `components/FluidGlassCard.tsx`**
- Detect mobile via `window.innerWidth < 1024` or `matchMedia`
- On mobile: skip `FluidGlass` (Three.js canvas) entirely
- Keep the `backdrop-filter: brightness() blur()` base layer — it's already there and provides readability
- Optionally add a simpler CSS-only glass effect fallback (just the blur + box-shadow borders)

**File: `sections/Hero.tsx`**
- `LiquidGlassCard` → on mobile, use the CSS-only version (no Three.js)
- The Hero card already has `max-sm:!relative` and restructured layout for mobile — just needs the 3D canvas disabled

**File: `sections/Projects.tsx`**
- `FluidGlassCard` → same mobile detection, skip 3D canvas

### Phase 2: Disable Matter.js Physics on Mobile

**File: `sections/tools.tsx`**
- On mobile (`isMobile`), don't render `<Gravity>` at all
- Instead, render tool pills as a simple CSS grid/flex layout
- Keep the bucket visual containers as simple styled divs
- This eliminates: Matter.js engine init, 23+ physics bodies, canvas render loop, mouse constraint, wall calculations

**File: `components/fancy/physics/gravity.tsx`**
- No changes needed — just stop rendering it on mobile

### Phase 3: Fix LogoCarousel Re-renders

**File: `components/ui/logo-carousel.tsx`**
- Replace `setInterval(updateTime, 100)` with `setInterval(updateTime, 1000)` or use CSS animation for the cycling
- Or better: use `useRef` + `requestAnimationFrame` for time tracking, only update React state when the visible logo actually changes (every 4000ms, not every 100ms)
- This reduces re-renders from 10/second to ~0.25/second per column

### Phase 4: Optimize GradualBlur for Mobile

**File: `app/layout.tsx`**
- On mobile: reduce `divCount` from 5 to 2, or disable `GradualBlur` entirely
- Or conditionally render: only show on desktop where GPU headroom exists

### Phase 5: Optimize Video Backgrounds

**File: `app/layout.tsx`**
- On mobile: replace `<video>` with a static `<img>` or CSS gradient fallback
- Or add `preload="none"` and only autoplay on desktop
- Reduces mobile GPU decode from 2 video streams to 0

### Phase 6: Optimize InteractiveGrid for Mobile

**File: `components/InteractiveGrid.tsx`**
- On mobile: disable the canvas animation loop (no `requestAnimationFrame`)
- Keep the CSS grid background (pure CSS, no canvas needed)
- Remove `mousemove` listener on mobile (no mouse)
- This is a layout wrapper in `layout.tsx` — affects entire page

### Phase 7: Simplify SVG Filters on Mobile

**Files: `components/GlassSurface.tsx`, `components/GlassCube.tsx`**
- On mobile: skip SVG filter pipeline, use simple `backdrop-filter: blur()` only
- The SVG `feDisplacementMap` chains are extremely expensive on mobile GPUs

### Phase 8: Icon Loading Optimization

**File: `sections/tools.tsx`**
- Tool pills fetch 23 icons from `cdn.simpleicons.org` — add `loading="lazy"` and consider inlining critical icons
- Or use a single sprite/SVG for the tools section
